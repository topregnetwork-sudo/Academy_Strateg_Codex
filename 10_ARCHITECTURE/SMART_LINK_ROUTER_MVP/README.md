# Smart Link Router MVP — standalone Apps Script Web App

[ЭТАП] `AS-SMART-LINK-ROUTER-MVP-001`

## Статус

[ПОДТВЕРЖДЕНО] Код подготовлен локально как отдельный Apps Script проект.

[ЗАПРЕТ] Этот каталог не является патчем production-генератора. Код не развёрнут и не подключён к боевым ссылкам.

## Файлы Apps Script

- `Code.gs` — проверка `link_id`, построение маршрута и опциональная запись события.
- `Router.html` — промежуточная HTML-страница, получение `user_agent` и безопасная ссылка `Продолжить`.

## Script Properties — только после отдельных решений

- `SITE_BASE_URL` — HTTPS-адрес сайта без `/index.html`, query и fragment.
- `EVENT_LOG_ENABLED` — строго `true` для включения записи; иначе журнал не изменяется.
- `EVENT_LOG_SPREADSHEET_ID` — ID отдельно утверждённой таблицы журнала.
- `EVENT_LOG_SHEET_NAME` — имя заранее созданного листа с точным заголовком схемы.

Значения не хранятся в репозитории. Код не создаёт таблицу или лист, не меняет заголовки и не пишет события, пока `EVENT_LOG_ENABLED` не равен `true`.

## Техническое ограничение MVP

Apps Script `HtmlService` возвращает HTML, а не настраиваемый HTTP 302, и sandbox ограничивает автоматическую навигацию верхнего окна. Поэтому после серверной проверки открывается минимальная страница, которая вызывает `recordClick` через `google.script.run`, а затем показывает ссылку `Продолжить` с `target="_top"` через `<base target="_top">`. Через 1,5 секунды ссылка показывается даже при зависании журнала, чтобы отказ журнала не блокировал воронку полностью.

[РИСК] Это дополнительное действие пользователя. До deploy нужно решить, приемлем ли такой UX или Router следует реализовать на платформе с серверным HTTP 302.
