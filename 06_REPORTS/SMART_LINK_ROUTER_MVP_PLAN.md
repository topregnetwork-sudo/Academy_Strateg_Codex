# Smart Link Router MVP Plan

[ЭТАП] `AS-SMART-LINK-ROUTER-MVP-001`

## [ЦЕЛЬ]

Добавить отдельный измеримый слой перехода между именной ссылкой Бэтмана и страницей сайта, не меняя production-генератор и текущую воронку до подтверждённого теста.

Участок воронки: источник / рекомендация / стажёр → собственник бизнеса → бизнес-тест.

## [ПОДТВЕРЖДЕНО]

- Входной идентификатор имеет целевой вид `btm_XXXXXX_hr_invite`.
- `btm_id` должен стать `btm_XXXXXX`.
- `link_type` и `source_id` для MVP равны `hr_invite`.
- Target должен вести на `SITE_BASE_URL/index.html` с параметрами `ref`, `source_id`, `link_type`, `click_id`.
- Код должен быть отдельным Apps Script Web App и пока не должен быть развёрнут.

## [АРХИТЕКТУРА MVP]

1. `doGet(e)` получает только `link_id` из query string.
2. Сервер проверяет `link_id` по шаблону `^btm_\d{6}_hr_invite$`.
3. Сервер генерирует `click_id` вида `clk_<timestamp_ms>_<uuid32>`.
4. `SITE_BASE_URL` читается из Script Properties; в коде и GitHub значения нет.
5. Сервер строит `target_url` из доверенных полей.
6. HTML-страница получает `navigator.userAgent`, асинхронно вызывает `recordClick` и затем показывает ссылку `Продолжить` с target `_top`.
7. Сервер повторно проверяет `link_id` и `click_id`, заново строит все поля и при включённом журнале добавляет строку.
8. Через 1,5 секунды ссылка показывается даже без ответа журнала, чтобы проблема логирования не блокировала воронку полностью.

[РИСК] `HtmlService` работает в iframe sandbox и не предоставляет надёжный автоматический HTTP redirect верхнего окна. Этот MVP требует дополнительного действия пользователя. Если бизнес-критичен автоматический 302, Router нужно размещать на другом HTTP runtime.

## [КОНФИГУРАЦИЯ]

Только Script Properties, без значений в репозитории:

- `SITE_BASE_URL`;
- `EVENT_LOG_ENABLED`;
- `EVENT_LOG_SPREADSHEET_ID`;
- `EVENT_LOG_SHEET_NAME`.

Event-log выключен по умолчанию. Код не создаёт таблицы, листы или заголовки автоматически.

## [СТАТУСЫ]

- `redirect_ready` — формат и конфигурация корректны, клиенту подготовлен переход;
- `invalid_link` — `link_id` не прошёл строгую проверку;
- `config_error` — отсутствует или некорректен `SITE_BASE_URL`.

Ошибка записи журнала возвращается отдельно как `log_error`, чтобы не подменять статус маршрута.

## [МАСШТАБИРОВАНИЕ]

После подтверждения MVP новые типы ссылок должны добавляться через отдельную таблицу разрешённых маршрутов, а не через ослабление регулярного выражения. Интеграции с amoCRM, n8n, Telegram и Calendar не входят в этот этап.

## [НЕ ПРОВЕРЕНО]

- реальный `SITE_BASE_URL`;
- фактическое место event-log;
- Apps Script scopes и режим выполнения Web App;
- реальный переход в браузере после deploy;
- запись одной тестовой строки.

## [ЗАПРЕТ]

Не применять generator diff, не деплоить, не менять production Apps Script, 3000 строк, `business_test_main`, `AS-BOT__00_MAIN_ROUTER`, n8n, Telegram или HR-форму.

## [ОДИН СЛЕДУЮЩИЙ ШАГ]

Утвердить `SITE_BASE_URL` и отдельное место event-log без deploy и без изменения строк.
