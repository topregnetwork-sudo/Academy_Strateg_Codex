# Smart Link Router Event Log Schema

[ЭТАП] `AS-SMART-LINK-ROUTER-MVP-001`

## [РЕШЕНИЕ]

Event-log является отдельным журналом Router. Он не должен размещаться в HR-таблице или боевом листе без отдельного подтверждения.

Точный порядок колонок обязателен:

| № | Поле | Тип / формат | Источник | Назначение |
|---:|---|---|---|---|
| 1 | `timestamp` | ISO 8601 UTC | сервер | Время попытки записи события |
| 2 | `click_id` | `clk_<13 цифр>_<32 hex>` | сервер | Уникальный ID перехода |
| 3 | `link_id` | `btm_XXXXXX_hr_invite` | запрос, затем проверка | Именная smart-ссылка |
| 4 | `btm_id` | `btm_XXXXXX` | сервер | ID Бэтмана |
| 5 | `link_type` | `hr_invite` | сервер | Тип ссылки |
| 6 | `source_id` | `hr_invite` | сервер | Источник для сайта |
| 7 | `target_url` | HTTPS URL | сервер | Сформированный адрес назначения |
| 8 | `status` | enum | сервер | Результат маршрутизации |
| 9 | `user_agent` | text, до 500 символов | браузер, недоверенное поле | Технический контекст клиента |
| 10 | `error` | безопасный код | сервер | Код ошибки без секретов и stack trace |

## [ДОПУСТИМЫЕ STATUS]

- `redirect_ready`;
- `invalid_link`;
- `config_error`.

Сбой самого журнала передаётся вызывающей странице как `log_error` и пишется только в Apps Script console безопасным кодом. Router не создаёт рекурсивную строку `log_error` в недоступном журнале.

## [ПРАВИЛА ЦЕЛОСТНОСТИ]

- Заголовок листа должен заранее существовать и строго совпадать с указанным порядком.
- При несовпадении заголовка запись запрещается; код не исправляет лист автоматически.
- Поля маршрута строятся повторно на сервере и не принимаются от браузера.
- Значения, начинающиеся с `=`, `+`, `-` или `@`, экранируются перед `appendRow` для защиты от формул.
- Параллельные записи сериализуются Script Lock.
- При выключенном `EVENT_LOG_ENABLED` таблица не открывается и не изменяется.

## [ПРИВАТНОСТЬ И ДОСТУП]

`user_agent` может помогать диагностике, но остаётся недоверенным техническим идентификатором. До включения журнала нужно утвердить владельца, список имеющих доступ и срок хранения. Не добавлять IP, cookies, Telegram ID, ФИО, телефон, email или другие данные кандидатов в эту схему.

## [ОДИН СЛЕДУЮЩИЙ ШАГ]

Выбрать отдельную Spreadsheet/лист и политику хранения, не создавая их до подтверждения.
