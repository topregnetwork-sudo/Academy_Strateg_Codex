# AS-SELLER-RAZBOR-MVP__SELLER_FUNNEL_REPORT

## [SELLER FUNNEL]

Полная воронка продавца / эксперта под Zoom-разборы:

1. Продавец / эксперт Академии Стратег уже заведён в таблице `Разборщики`.
2. Продавец пишет в Telegram bot команду `/seller`.
3. Router проверяет `telegram_username` / `telegram_chat_id` в листе `Разборщики`.
4. Если seller найден, bot подтверждает роль:
   `Вы подключены как эксперт Академии Стратег`.
5. Seller пишет `/availability 2026-07-15 10:00 12:00 45`.
6. Router создаёт окно доступности в `Разборщики_Доступность`.
7. Router генерирует свободные слоты в `Разборы_Слоты`.
8. Собственник проходит owner flow и выбирает свободный слот.
9. Router повторно проверяет слот и создаёт booking.
10. Booking пишется в `Разборы_Записи`.
11. Pending reminders пишутся в `Разборы_Напоминания`.
12. Seller получает уведомление о новой записи.
13. Seller открывает Zoom по ссылке из записи / слота / карточки разборщика.
14. Seller проводит разбор результата бизнес-теста.
15. Seller пишет `/schedule`, чтобы видеть ближайшие записи.
16. После разбора seller пишет `/comment BOOKING_ID проведён комментарий`.
17. Итог разбора фиксируется в `Разборы_Комментарии`.
18. Статус и следующий шаг ведут к продаже обзорного курса стратегического управления за `60 000 ₽`.

## [SELLER COMMANDS]

`/seller`

- включает seller-режим;
- проверяет, что пользователь есть в `Разборщики`;
- записывает `telegram_chat_id`;
- подтверждает роль эксперта.

`/availability`

- MVP-формат:

```text
/availability 2026-07-15 10:00 12:00 45
```

- создаёт окно доступности;
- генерирует слоты длительностью 30 / 45 / 60 минут;
- пишет слоты со статусом `свободен`.

`/schedule`

- показывает ближайшие активные записи seller;
- берёт данные из `Разборы_Записи`.

`/comment`

- MVP-формат:

```text
/comment BOOKING_ID проведён комментарий
```

- создаёт строку в `Разборы_Комментарии`.

`/help`

- в import-ready workflow пока не выделен отдельной веткой;
- должен показывать seller-команды и формат ввода.

## [ZOOM]

Где хранится `zoom_link`:

- `Разборщики.zoom_link` — основной источник ссылки эксперта;
- `Разборы_Слоты.zoom_link` — ссылка копируется в слот;
- `Разборы_Записи.zoom_link` — ссылка фиксируется в booking.

Как seller получает Zoom:

- при новой записи seller получает уведомление с Zoom-ссылкой;
- в текущем import-ready workflow это текстовая ссылка;
- целевой вариант: inline-кнопка `Открыть Zoom`.

Как owner получает Zoom:

- после успешного booking owner получает дату, время, эксперта и Zoom-ссылку;
- Zoom не хардкодится, берётся из таблицы.

Что делать с `backup_meeting_link`:

- хранить в `Разборщики`, `Разборы_Слоты`, `Разборы_Записи`;
- использовать, если основная Zoom-ссылка недоступна;
- позже можно добавить логику fallback в bot-router.

Что пока не проверено:

- тариф Zoom;
- можно ли проводить параллельные встречи;
- права ведущего;
- ограничения персональной комнаты;
- нужен ли отдельный Zoom/meeting link на каждого эксперта.

## [SELLER SCHEDULE]

Команда:

```text
/schedule
```

Как работает:

- bot ищет seller по `telegram_chat_id` или `telegram_username`;
- читает `Разборы_Записи`;
- показывает ближайшие записи, где `seller_id` совпадает;
- исключает закрытые статусы: `completed`, `no_show`, `sale_done`, `sale_lost`.

MVP-формат вывода:

```text
booking_... — 2026-07-15T10:00:00 — owner TEST_owner_001 — booked
```

Целевой формат:

```text
Ближайшие разборы:

1. [дата] [время] МСК
   booking_id: [booking_id]
   собственник: [ФИО]
   компания: [организация]
   результат: [result_url]
```

## [SELLER COMMENT]

После разбора seller должен фиксировать:

- статус разбора;
- главная боль;
- квалификация по деньгам;
- интерес к курсу;
- возражения;
- рекомендованный триггер;
- комментарий;
- следующее действие;
- дата следующего контакта.

В текущем import-ready workflow MVP-команда:

```text
/comment BOOKING_ID проведён комментарий
```

Сейчас она заполняет базовые поля:

- `comment_id`;
- `created_at`;
- `booking_id`;
- `owner_id`;
- `seller_id`;
- `статус_разбора`;
- `комментарий_разборщика`;
- `следующее_действие`.

Поля для расширенного seller-сценария уже есть в таблице `Разборы_Комментарии`:

- `главная_боль`;
- `квалификация_по_деньгам`;
- `интерес_к_курсу`;
- `возражения`;
- `рекомендованный_триггер`;
- `дата_следующего_контакта`.

## [SELLER TABLES]

Участвующие листы:

`Разборщики`

- карточка seller;
- `seller_id`;
- `telegram_username`;
- `telegram_chat_id`;
- `zoom_link`;
- `backup_meeting_link`;
- `meeting_provider`;
- статус и специализация.

`Разборщики_Доступность`

- окно доступности seller;
- дата;
- start/end;
- длительность слота;
- количество созданных слотов.

`Разборы_Слоты`

- конкретные свободные/занятые слоты;
- `slot_id`;
- `seller_id`;
- `zoom_link`;
- `статус`;
- `owner_id`;
- `booking_id`.

`Разборы_Записи`

- booking после выбора owner;
- `booking_id`;
- `owner_id`;
- `seller_id`;
- `slot_id`;
- `result_url`;
- `zoom_link`;
- статусы reminders;
- итог/следующее действие.

`Разборы_Комментарии`

- итог seller после разбора;
- боль, деньги, интерес, возражения, триггер, следующий контакт.

`Разборы_Напоминания`

- pending reminders для owner и seller;
- в MVP строки создаются, авто-отправка ещё не включена.

`Логи_SELLER_BOT`

- целевой лист для seller events/errors;
- в текущем import-ready workflow отдельная запись логов seller ещё не включена.

## [ТЕСТ SELLER FLOW]

Что проверить после импорта workflow:

1. Написать `/seller` от `@TopregStrateg`.
2. Проверить, что `telegram_chat_id` записался в `Разборщики`.
3. Написать:

```text
/availability 2026-07-15 10:00 12:00 45
```

4. Проверить, что создалась строка в `Разборщики_Доступность`.
5. Проверить, что создались слоты в `Разборы_Слоты`.
6. Пройти owner flow и выбрать один слот.
7. Проверить, что seller получил уведомление.
8. Написать `/schedule`.
9. Проверить, что `/schedule` показал запись.
10. Написать:

```text
/comment [booking_id] проведён тестовый комментарий
```

11. Проверить, что `Разборы_Комментарии` обновились.

## [НЕ ХВАТАЕТ]

- Авто-напоминания: сейчас создаются pending rows, но отдельный cron/reminder workflow ещё не включён.
- Кнопка `Открыть Zoom`: в текущем import-ready workflow Zoom пока текстом.
- Проверка тарифа Zoom: параллельные встречи, права ведущего, персональные комнаты.
- Отдельный seller bot: сейчас используется общий `@Akademya_Strateg_bot`.
- Полноценный post-sale сценарий: статусы продажи есть, но детальная логика после разбора ещё не реализована.
- Полноценный пошаговый `/comment`: сейчас MVP-формат однострочный.
- Seller logs: лист `Логи_SELLER_BOT` есть, но отдельные log append nodes пока не включены.

## [РЕШЕНИЕ]

Для тестового прогона используем один общий bot:

```text
@Akademya_Strateg_bot
```

Архитектурное решение:

- один Telegram Trigger;
- один import-ready router:
  `AS-RAZBOR-BOT__00_MAIN_ROUTER_IMPORT_READY`;
- owner и seller роли разделяются внутри router;
- seller определяется по `telegram_username` / `telegram_chat_id`;
- позже seller-контур можно вынести в отдельного бота без переделки таблиц.

## [ЗАПРЕТ]

Не создавать второй активный Telegram Trigger на тот же bot.

Нельзя одновременно включать два workflow, которые слушают:

```text
@Akademya_Strateg_bot
```

Также нельзя:

- менять `AS-BOT__00_MAIN_ROUTER`;
- менять HR/Batman bot;
- менять текущий business-test webhook;
- хранить Telegram token в workflow JSON;
- показывать owner данные Бэтмана;
- показывать Бэтману телефоны/результаты owner без отдельного решения.
