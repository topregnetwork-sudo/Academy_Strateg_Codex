# Smart Link Router MVP Test Plan

[ЭТАП] `AS-SMART-LINK-ROUTER-MVP-001`

## [ОБЛАСТЬ]

План проверяет только отдельный Router. Production-генератор, `business_test_main`, `AS-BOT__00_MAIN_ROUTER`, n8n, Telegram и HR-форма исключены.

## [УРОВЕНЬ 1] Локальная статическая проверка — разрешено сейчас

1. Проверить синтаксис `Code.gs` в изолированной среде с заглушками Apps Script API.
2. Проверить строгий шаблон `link_id`.
3. Проверить извлечение `btm_id`.
4. Проверить формат и уникальность структуры `click_id`.
5. Проверить точный порядок query-параметров и URL-encoding.
6. Проверить отказ при пустом или некорректном `SITE_BASE_URL`.
7. Проверить точный порядок десяти колонок event-log.
8. Проверить экранирование formula-like значений.
9. Проверить наличие client-side user-agent capture, `google.script.run` и fallback-показа ссылки `Продолжить`.
10. Выполнить поиск секретов и запрещённых боевых значений в новых файлах.

## [МАТРИЦА ROUTER]

| Сценарий | Вход | Ожидаемый результат |
|---|---|---|
| Валидный | `btm_123456_hr_invite` | `btm_id=btm_123456`, `redirect_ready`, точный target URL |
| Нет параметра | пусто | `invalid_link`, перехода нет |
| Мало цифр | `btm_12345_hr_invite` | `invalid_link` |
| Лишние цифры | `btm_1234567_hr_invite` | `invalid_link` |
| Неверный тип | `btm_123456_other` | `invalid_link` |
| Дополнительный текст | `xbtm_123456_hr_invite` | `invalid_link` |
| Нет SITE_BASE_URL | валидный `link_id` | `config_error`, перехода нет |
| Некорректный SITE_BASE_URL | HTTP/query/fragment | `config_error`, перехода нет |
| Event-log выключен | валидный маршрут | ссылка показывается, таблица не открывается |
| Event-log недоступен | валидный маршрут | ссылка показывается, возвращается безопасный `log_error` |

## [РЕЗУЛЬТАТ УРОВНЯ 1]

[ПОДТВЕРЖДЕНО] Изолированный локальный набор проверок завершён с результатом `SMART_LINK_ROUTER_LOCAL_TESTS_OK`.

Проверены валидный маршрут, шесть невалидных вариантов `link_id`, отсутствие/некорректность `SITE_BASE_URL`, формат `click_id`, точный порядок event-log, защита от formula injection и обязательные элементы `Router.html`. Временный тестовый harness удалён и в проект не входит.

## [УРОВЕНЬ 2] Apps Script test deployment — пока запрещено

После отдельного разрешения:

1. Создать новый standalone Apps Script проект.
2. Внести только утверждённые Script Properties.
3. Развернуть отдельный test deployment с утверждённым режимом доступа.
4. Проверить invalid/config cases без изменения боевых строк.
5. Проверить одну искусственную тестовую запись в отдельном event-log.

## [УРОВЕНЬ 3] Тест одной строки hr_invite — отдельное разрешение

До теста зафиксировать:

- конкретную строку и текущее значение;
- backup до изменения;
- Web App URL тестового deployment;
- ожидаемый target URL и event-log row;
- время теста и ответственного;
- точную команду rollback.

Изменить ровно одну утверждённую строку, выполнить один переход, сверить target и журнал, затем принять отдельное решение: rollback или сохранение тестового значения. Массовое изменение 3000 строк не входит в MVP.

## [КРИТЕРИИ ПРИЁМКИ]

- неправильные `link_id` не перенаправляются;
- правильный `link_id` даёт точный `btm_id`, type/source и новый `click_id`;
- target строится только сервером из Script Properties и проверенных полей;
- схема журнала совпадает по десяти колонкам;
- отказ event-log не скрывает валидную ссылку `Продолжить`;
- в репозитории, HTML и журнале нет секретов;
- production-слои не изменены.

## [ОДИН СЛЕДУЮЩИЙ ШАГ]

Завершить уровень 1 локально; уровни 2 и 3 не начинать без отдельных разрешений.
